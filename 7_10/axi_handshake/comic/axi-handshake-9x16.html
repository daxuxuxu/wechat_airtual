1握手基本规则：两个独立方向
每个通道一对信号：valid 发送方驱动=「我有合法数据」；ready 接收方驱动=「我能收」。两者同一拍都为高，才完成一次传输。
↑ 黄框：两者同高 = 完成一次传输
valid 和 ready 完全独立！
发送方可先拉 valid 等接收方，
接收方也可先拉 ready 等发送方，
谁先谁后都不影响正确性 ✅
2铁律：VALID 不能等 READY
ready 允许依赖对方当前的 valid；但 valid 一旦拉高绝不能因对方没 ready 而撤回或延迟。方向反了就是死锁。
✅ 合法
模块A
上游→模块B
下游
valid → ready
单向因果，不成环
❌ 非法
模块A⇄模块B
A.valid 等 B.ready
B.ready 等 A.valid
互相等 → 卡死
一句伪代码看清对错 👇 valid 只看「我有没有数据」，绝不看对方 ready。
// ❌ 错：valid 依赖对方 ready，构成组合环路 assign up_valid = has_data && dn_ready; // ✅ 对：valid 只看自己是否真有数据 assign up_valid = has_data; // ready 看对方当前 valid（这个方向允许） assign dn_ready = can_accept && up_valid;
3反压是怎么逐级传导的
末端某级收不下 → 拉低自己的 ready；上游发不出、数据滞留，缓冲打满后它的 ready 也被拉低，压力继续往上传。
发起端→ 级1→ 级2→ 级3→ 目标端
ready=0
← 反压 (ready 拉低) 逐级向上
🎯 反压方向永远和数据流相反。所以「上游突然发不出」时，别盯上游，顺着数据流往下游追第一个拉低 ready 的源头。
原来如此！
卡死先往下游找，
而不是死磕上游逻辑～ 💡

5验证值得关注的维度
① 握手方向静态检查：valid 不得组合依赖同方向 ready，跨模块间接依赖易漏，需人工复核。
② 反压传导链完整：下游持续拉低 ready，确认逐级上传、不被某级「吞掉」。
③ 极限边界：反压拉满 / 数据源枯竭两种极端下，不死锁、不丢数据。
④ 缓冲深度匹配：时序生成 ready 时，缓冲要够吸收「提前放行」的时间差。
⑤ 握手稳定性：valid 拉高到被接收前，数据与 valid 不得变化。
一条因果律，管住整条流水线
valid 只由「我有数据」决定，
ready 才可由「对方有数据」决定，
方向反过来 = 死锁。
